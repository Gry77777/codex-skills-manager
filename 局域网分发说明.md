# Codex 技能管理器局域网分发说明

## 生成分发包

```powershell
npm run dist
```

生成结果在 `release/` 目录：

- `Codex-Skills-Manager-版本号-x64.exe`：便携版，直接双击运行。
- `Codex-Skills-Manager-版本号-x64.zip`：压缩包，适合放到共享目录或发给局域网用户。

## 给其他人使用

把 `release/` 里的 `.exe` 或 `.zip` 发给对方即可。对方不需要安装 Node.js，也不需要安装 npm。

应用启动后会扫描当前 Windows 用户自己的目录：

- `%USERPROFILE%\.codex\skills`
- `%USERPROFILE%\.agents\skills`
- `%APPDATA%\CodexSkillsManager\imported-skills`

因此每个人看到的是自己电脑上的 skills，不会导入或覆盖打包者电脑上的 skills。

## 注意

- 当前包已经带自定义图标，图标源文件在 `build/icon-source.png`，Windows 图标在 `build/icon.ico`。
- 首次运行时 Windows SmartScreen 可能提示未知发布者，这是因为当前包没有代码签名证书。
- 一键开关会通过 `SKILL.md` / `SKILL.md.disabled` 文件名切换真实启用状态。
- 一键修复不会删除 skill 目录；冲突文件会改成备份名。

## 代码签名是什么

代码签名是给 `.exe` 加数字证书。签名后，Windows 可以显示发布者名称，并验证文件分发后没有被篡改。

没有代码签名时，软件仍然可以运行，但 Windows 可能显示“未知发布者”或 SmartScreen 提示。正式给更多人分发时，建议购买或申请 Windows 代码签名证书，然后配置 `electron-builder` 签名。
