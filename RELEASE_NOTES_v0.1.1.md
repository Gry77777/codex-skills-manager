# Codex Skills Manager v0.1.1

这是一次体验和稳定性更新，重点改进 AI 识别、导入后的自动缓存，以及技能广场空结果提示。

## 主要更新

- 导入本地文件夹、GitHub 技能或技能广场技能后，会自动加入后台 AI 识别队列。
- 后台 AI 识别只写入本机缓存，不会自动打开、关闭或执行任何 skill。
- AI 识别按批处理，避免一次请求过大。
- AI 输出解析更稳定，支持纯 JSON、Markdown JSON 代码块、前后带说明文字、轻微尾逗号等常见模型输出。
- OpenAI 兼容 API 会优先请求 JSON 模式；如果中转站或模型不支持，会自动退回普通模式。
- 批量 AI 识别失败提示会按原因聚合，不再重复刷同一句错误。
- 技能广场来源卡片会显示本地已索引数量。
- 当某个来源没有本地索引结果时，界面会明确提示，并提供“扫描这个来源”按钮。
- 技能卡片排版更紧凑，名称、来源、用途和开关更清晰。

## 使用说明

- 如果已经配置并启用 AI API，导入 skill 后会自动后台识别并缓存中文用途、标签、风险和启用建议。
- 如果没有配置 AI API，导入仍然正常完成，只是不会自动调用外部模型。
- 技能广场里的“已索引”是快速本地索引结果；如果某个来源为 0，可以点击“扫描来源”做深度识别。

## 下载

- `Codex-Skills-Manager-0.1.1-x64.exe`：Windows x64 便携版，直接运行。
- `Codex-Skills-Manager-0.1.1-x64.zip`：压缩包，适合局域网分发。

## 注意

- 当前版本仍未代码签名，Windows 可能提示未知发布者。
- 打开/关闭 skill 会真实切换 `SKILL.md` 与 `SKILL.md.disabled`。
- 已经运行中的 Codex 会话可能不会立即感知 skill 变更，建议开启新会话或重启 Codex。
- 本项目不是 OpenAI 或 Codex 官方产品。

## 发布前验证
